## 需求提交流程

### 1. 需求提交格式

请在 GitHub Issue 中提交需求，标题格式为：  
```
需求：[具体需求描述]
```

**示例**：  
```
需求：添加评论功能
```

### 2. Bug 提交格式

请在 GitHub Issue 中提交 Bug，标题格式为：  
```
bug：[具体问题描述]
```

**示例**：  
```
bug：首页加载缓慢
```

## 提交要求

### 需求提交要求
- 需求描述清晰明确，说明具体功能点
- 如有参考示例，请提供相关链接
- 说明需求的使用场景和价值

### Bug 提交要求
- **必须提供截图**：展示问题的具体表现
- **详细步骤**：描述如何复现问题
- **环境信息**：浏览器版本、操作系统等
- **预期行为**：说明正常情况下应该如何工作

## 处理流程

1. 需求提交后，我们会进行评估
2. **合理的需求**会被考虑加入开发计划
3. Bug 提交后，我们会尽快分析并修复
4. 处理结果会在 Issue 中回复

## 注意事项

- 请确保提交的需求或 Bug 是项目相关的
- 避免重复提交相同的内容
- 尊重其他贡献者，保持友好沟通

感谢您对项目的关注和支持！
